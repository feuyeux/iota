## 9. 核心设计原则实现

### 9.1 架构原则
- ✅ 所有后端以 subprocess 方式运行
- ✅ 不引入 vendor 内部 SDK
- ✅ 不 import vendor 源码
- ✅ 每个后端保留原生 stdio 协议
- ✅ Engine 内 adapter 负责协议适配
- ✅ 不直接执行后端 native tool
- ✅ 共享 MCP Server 由 Iota MCP Router 管理

### 9.2 统一大模型配置
- ✅ `IOTA_BASE_URL` / `IOTA_MODEL` / `IOTA_AUTH_TOKEN` 环境变量
- ✅ 所有后端共享同一组配置
- ✅ Adapter 负责映射为后端原生参数

### 9.3 事件系统
- ✅ 事件序号单调递增
- ✅ 先持久化再 yield
- ✅ 多路复用支持
- ✅ Register-before-replay 机制

### 9.4 工具执行权归属
- ✅ 后端 native 工具由后端自身执行
- ✅ MCP 工具由 Iota MCP Router 代理
- ✅ Iota 内部工具由 Engine 执行
